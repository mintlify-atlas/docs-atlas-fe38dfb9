## Create Custody Chain

Create a new custody chain for equipment tracking and service documentation.

<CodeGroup>
```bash cURL
curl -X POST https://api.mantis.com/api/workorders/custody_chain/create/ \
  -H "Authorization: Bearer YOUR_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "technical_id": 5,
    "vehicle_id": 3,
    "project_id": 10,
    "resources": [12, 15, 18],
    "activity_date": "2024-01-15",
    "location": "Guayaquil Plant",
    "start_time": "08:00:00",
    "end_time": "16:00:00",
    "duration_hours": 8.0,
    "total_gallons": 1500.0,
    "black_water": true,
    "have_logistic": "YES",
    "contact_name": "Carlos Mendez",
    "dni_contact": "0912345678"
  }'
```
</CodeGroup>

### Request Body

<ParamField body="technical_id" type="integer" required>
  ID of the technical/operator assigned to this custody chain
</ParamField>

<ParamField body="vehicle_id" type="integer" required>
  ID of the vehicle used for the service
</ParamField>

<ParamField body="project_id" type="integer" required>
  ID of the project. The system will automatically link to the most recent active sheet.
</ParamField>

<ParamField body="resources" type="array" required>
  Array of project resource item IDs to include in this custody chain. Must contain at least one resource.
</ParamField>

<ParamField body="consecutive" type="integer">
  Consecutive number (auto-generated if not provided)
</ParamField>

<ParamField body="activity_date" type="string">
  Date when the activity occurred (YYYY-MM-DD)
</ParamField>

<ParamField body="issue_date" type="string">
  Date when the custody chain was issued (YYYY-MM-DD)
</ParamField>

<ParamField body="status" type="string" default="DRAFT">
  Status of the custody chain: `DRAFT` or `CLOSE`
</ParamField>

<ParamField body="start_time" type="string">
  Service start time (HH:MM:SS format)
</ParamField>

<ParamField body="end_time" type="string">
  Service end time (HH:MM:SS format)
</ParamField>

<ParamField body="duration_hours" type="number">
  Duration in hours
</ParamField>

<ParamField body="location" type="string">
  Service location
</ParamField>

<ParamField body="contact_name" type="string">
  Name of the contact person at the service location
</ParamField>

<ParamField body="dni_contact" type="string">
  DNI/ID of the contact person
</ParamField>

<ParamField body="contact_position" type="string">
  Position/role of the contact person
</ParamField>

<ParamField body="date_contact" type="string">
  Date when contact signed (YYYY-MM-DD)
</ParamField>

<ParamField body="driver_name" type="string">
  Name of the driver
</ParamField>

<ParamField body="dni_driver" type="string">
  DNI/ID of the driver
</ParamField>

<ParamField body="driver_position" type="string">
  Position of the driver
</ParamField>

<ParamField body="driver_date" type="string">
  Date when driver signed (YYYY-MM-DD)
</ParamField>

<ParamField body="total_gallons" type="number" default="0">
  Total gallons processed
</ParamField>

<ParamField body="total_barrels" type="number" default="0">
  Total barrels processed
</ParamField>

<ParamField body="total_cubic_meters" type="number" default="0">
  Total cubic meters processed
</ParamField>

<ParamField body="black_water" type="boolean" default="false">
  Whether black water was processed
</ParamField>

<ParamField body="grey_water" type="boolean" default="false">
  Whether grey water was processed
</ParamField>

<ParamField body="clean_water" type="boolean" default="false">
  Whether clean water was processed
</ParamField>

<ParamField body="activated_sludge" type="boolean" default="false">
  Whether activated sludge was processed
</ParamField>

<ParamField body="treated_wastewater" type="boolean" default="false">
  Whether treated wastewater was processed
</ParamField>

<ParamField body="organic_grease" type="boolean" default="false">
  Whether organic grease was processed
</ParamField>

<ParamField body="have_logistic" type="string" default="NA">
  Logistics status: `YES`, `NO`, or `NA`
</ParamField>

<ParamField body="notes" type="string">
  Additional notes or observations
</ParamField>

### Response

<ResponseField name="success" type="boolean">
  Indicates if the custody chain was created successfully
</ResponseField>

<ResponseField name="message" type="string">
  Success or error message
</ResponseField>

<ResponseField name="data" type="object">
  Created custody chain information
  
  <Expandable title="Data Object">
    <ResponseField name="custody_chain" type="object">
      Main custody chain data with all submitted fields
    </ResponseField>
    
    <ResponseField name="technical" type="object">
      Technical/operator information (id, first_name, last_name)
    </ResponseField>
    
    <ResponseField name="vehicle" type="object">
      Vehicle information (id, no_plate, brand, model)
    </ResponseField>
    
    <ResponseField name="details" type="array">
      Array of custody chain detail items for each resource
      
      <Expandable title="Detail Item">
        <ResponseField name="id" type="integer">
          Detail item ID
        </ResponseField>
        
        <ResponseField name="project_resource_id" type="integer">
          Associated project resource ID
        </ResponseField>
        
        <ResponseField name="code_equipment" type="string">
          Full equipment code
        </ResponseField>
        
        <ResponseField name="equipment" type="string">
          Equipment abbreviation
        </ResponseField>
      </Expandable>
    </ResponseField>
    
    <ResponseField name="details_count" type="integer">
      Number of resources included
    </ResponseField>
  </Expandable>
</ResponseField>

### Validation Rules

- Cannot create custody chains for projects without an active sheet
- Cannot create custody chains if the sheet status is `LIQUIDATED`, `INVOICED`, or `CANCELLED`
- Cannot create custody chains if the sheet is closed
- All resources must be active and belong to the specified project
- At least one resource must be selected

## Edit Custody Chain

Update an existing custody chain and its resource details.

<CodeGroup>
```bash cURL
curl -X PUT https://api.mantis.com/api/workorders/custody_chain/1/edit/ \
  -H "Authorization: Bearer YOUR_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "custody_chain": {
      "status": "CLOSE",
      "location": "Updated Location",
      "total_gallons": 2000.0,
      "meta": {
        "notes": "Service completed"
```

## Regenerate Equipment Codes

Regenerate equipment codes for all active custody chain details. Useful after equipment updates.

<CodeGroup>
```bash cURL
curl -X POST https://api.mantis.com/api/workorders/custody_chain/regenerate-codes/ \
  -H "Authorization: Bearer YOUR_TOKEN"
```
</CodeGroup>

### Response

<ResponseField name="success" type="boolean">
  Indicates if the regeneration was successful
</ResponseField>

<ResponseField name="message" type="string">
  Success message with summary
</ResponseField>

<ResponseField name="updated" type="integer">
  Number of custody chain details updated with new codes
</ResponseField>

<ResponseField name="skipped" type="integer">
  Number of details skipped (no equipment code available)
</ResponseField>

### Notes

- This endpoint processes all active custody chain details in the system
- Only updates details where a valid equipment code can be determined
- Equipment codes are extracted from the physical equipment associated with each project resource
